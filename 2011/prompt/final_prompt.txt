Key Pointer Concepts in This Lab
Notice how main.cpp declares a local Inventory inv; variable and passes its address &inv to all functions. Inside those functions, you access members using the arrow operator:

// In main.cpp:
Inventory inv;
initInventory(&inv, shopName, maxItems);  // pass address of inv
addItem(&inv, name, qty, price, weight);  // pass address of inv

// Inside the functions, 'inv' is a pointer (Inventory*):
inv->itemCount    // access member through pointer (arrow operator)
inv->items[i]     // access array element through pointer
&inv->items[i]    // get address of an array element (returns Item*)

// The itemPtrs[] array stores pointers into items[]:
inv->itemPtrs[i]           // an Item* that points somewhere into items[]
inv->itemPtrs[i]->price    // access a field via the pointer-of-pointer chain

The Item struct and Inventory struct are already defined in lab7.h:

struct Item {
    char name[MAX_NAME_LENGTH];   // item name (e.g., "Health Potion")
    int quantity;                  // number in stock
    double price;                 // price per unit in gold
    double weight;                // weight per unit in kg
};

struct Inventory {
    char shopName[MAX_NAME_LENGTH];  // name of the shop
    Item items[MAX_ITEMS];           // fixed-size array of Items (insertion order)
    Item* itemPtrs[MAX_ITEMS];       // array of pointers into items[] (used for sorting)
    int itemCount;                   // current number of distinct items
    int capacity;                    // user-chosen limit on distinct items (<= MAX_ITEMS)
};
Notice that Inventory has two parallel arrays: items[] stores the actual Item data in insertion order, and itemPtrs[] stores pointers into items[]. The pointer array allows sorting the display order without moving the underlying data.
